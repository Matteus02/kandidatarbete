================================================================================
  EIS ANALYZER — PROJECT STRUCTURE
================================================================================

Technology stack:
  - Vue 3 (Composition API, <script setup>)
  - TypeScript
  - Plotly.js  (plots)
  - ONNX Runtime Web  (AI inference in a web worker)
  - ml-levenberg-marquardt  (Levenberg-Marquardt optimiser)
  - PapaParse  (CSV parsing)
  - Vite  (build tool)

  src/types/
--------------------------------------------------------------------------------

eis.ts
  Shared TypeScript type definitions for the entire application.
    EisDataPoint  — one row of measurement data (freq, Re(Z), -Im(Z), |Z|, phase)
    Circuit, FitResult, KKResult, LocalStore  — data-model and result types

modelConfig.ts
  Single source of truth for AI model configuration.
    MODEL_URL         — path to the ONNX model file (served from /models/)
    KNOWN_CIRCUITS    — the circuit strings the model can predict
    N_POINTS          — number of log-spaced frequency points (60)
    CHANNEL_DEFS      — the 5 input channels fed to the CNN

workerProtocol.ts
  Message types shared between AIPanel.vue and the inference web worker.
    InferenceRequest  — sent TO the worker: { type: 'run', data: EisDataPoint[] }
    PredictionItem    — { circuit: string, confidence: number }
    InferenceResponse — received FROM the worker: result or error variant

fittingWorkerProtocol.ts
  Message types shared between useLMFitting.ts and the LM fitting web worker.
    SerializedNode    — JSON-serializable snapshot of a CircuitNode
    FitRequest        — sent TO the worker: serialized tree + data points
    FitResponse       — received FROM the worker: fitted parameter values or error

--------------------------------------------------------------------------------
  src/workers/
--------------------------------------------------------------------------------

eisInference.worker.ts
  Runs ONNX model inference in a separate thread so the UI stays responsive.
  Steps performed on each 'run' message:
    1. Sort data high → low frequency.
    2. Interpolate to a fixed 60-point log-spaced frequency grid.
    3. Build a [1, 5, 60] tensor with 5 channels:
         ch0  Re(Z)
         ch1  Im(Z)  (negated: file stores -Im(Z))
         ch2  log10(frequency grid)
         ch3  d(unwrapped phase) / d(log10 f)  — phase derivative
         ch4  d(log|Z|) / d(log10 f)  — Bode slope
    4. Per-channel z-score normalisation.
    5. Run inference via ONNX Runtime Web (WASM backend).
    6. Softmax the logits → confidence scores.
    7. Post { type: 'result', predictions } back to AIPanel.

lmFitting.worker.ts
  Runs Levenberg-Marquardt curve fitting in a separate thread.
  Receives a serialized CircuitNode tree and EIS data points, deserializes
  the tree, runs fitCircuit() from lmFitting.ts, and posts fitted parameter
  values back to the main thread.

--------------------------------------------------------------------------------
  src/utils/
--------------------------------------------------------------------------------

CircuitNode.ts
  The core data structure for the circuit tree.
  Class CircuitNode:
    id           — unique string label shown on the symbol (e.g. "R0", "CPE1")
    type         — ElementType: R | C | CPE | W | Wo | Ws | L | parallel | end | empty
    value        — primary parameter (resistance, capacitance, etc.)
    value2       — second parameter used by Wo and Ws (diffusion time constant τ)
    next/earlier — doubly-linked series chain pointers
    upperBranch/lowerBranch — parallel branch child pointers
  Methods:
    countLength() / countMaxLength()  — horizontal pixel span for SVG layout
    countHeight() / countMaxHeight()  — vertical pixel span for SVG layout
    removeNode()                      — detach from the tree

complexMath.ts
  Pure complex-number arithmetic. No Vue, no side effects.
  Exports:
    Complex           — type { re: number; im: number }
    add, mul, div     — arithmetic operations
    csqrt, cexp, ccosh, csinh  — complex transcendental functions

impedance.ts
  Maps physical element parameters + angular frequency ω to complex impedance Z.
  Exports:
    parallel(a, b)     — parallel combination Z = (Z1·Z2)/(Z1+Z2)
    zR, zC, zCPE, zW, zL, zWo, zWs  — per-element impedance formulas

circuitImpedance.ts
  Calculates Z(ω) for an entire CircuitNode tree by recursive traversal.
  Exports:
    zOfNode(node, ω)   — impedance of a single node or parallel block
    zOfChain(node, ω)  — total series impedance of a chain (follows .next)

circuitParser.ts
  Converts circuit strings into CircuitNode trees. Supports series elements
  inside parallel branches (e.g. "p(C0,R1-W0)").
  Input format:  "R0-p(CPE0,R1)-W0"
    - "-" separates series elements at the top level
    - "p(upper, lower)" describes a parallel block; each branch may itself
      be a series chain
    - element IDs start with the type letter (R, C, CPE, Wo, Ws, W, L)
  Exports:
    buildTreeFromString(circuitString)  — returns the root CircuitNode
    stringifyTree(root)                 — converts a tree back to a string

commonCircuits.ts
  List of predefined EIS circuits shown in the dropdown menu.
  Each entry has a name (user-facing label), circuit string, and description.
  Covers 12 common EIS topologies including Randles variants, two-time-constant
  models, and circuits with Warburg diffusion elements.

csvParser.ts
  Parses uploaded CSV files into EisDataPoint arrays using PapaParse.
  Handles two file formats:
    - BioLogic format: columns freq/Hz, Re(Z)/Ohm, -Im(Z)/Ohm, Phase(Z)/deg
    - Coin-cell format: columns freq/Hz, neg. Phase/°, Z/Ohm, Z'/Ohm, Z''/Ohm
  Automatically detects delimiter, header row, and column sign conventions.
  Negates "neg. Phase" values so all stored phase values use the same sign
  convention (negative for capacitive systems).

lmFitting.ts
  Levenberg-Marquardt curve fitting for EIS data.
  Fits circuit model parameters by minimising the residuals between measured
  and modelled impedance. Operates in log-space so parameters spanning many
  orders of magnitude (e.g. R ≈ 100 Ω, C ≈ 1e-6 F) have equal weight.

residuals.ts
  Computes normalised residuals between measured and modelled impedance.
    Real residual      = (Z'_meas - Z'_mod) / |Z_meas| × 100
    Imaginary residual = (Z''_meas - Z''_mod) / |Z_meas| × 100

chiSquared.ts
  Calculates the chi-squared goodness-of-fit statistic between measured
  and modelled Re(Z) and Im(Z) arrays.

kramersKronig.ts
  Performs a Kramers-Kronig consistency test on EIS data to assess data
  quality. Returns a KKResult with residuals and a pass/fail indicator.

exportUtils.ts
  File download utilities.
    downloadFile(filename, content, mimeType)  — triggers a browser download

--------------------------------------------------------------------------------
  src/composables/
--------------------------------------------------------------------------------

useCircuitTree.ts
  Vue composable. Owns all circuit tree state and every structural mutation.
  This is the single place where nodes are created, moved, or deleted.
  Returns:
    rootNode              — Ref<CircuitNode>  (the tree root)
    renderVersion         — Ref<number>  (incremented on each change; used as SVG :key)
    collectNodes(root)    — returns all leaf nodes with editable parameters
    resetCounters()       — resets ID counters (called after loading a new circuit)
    handleNodeDrop(...)   — inserts a new node before / replacing / after a target
    insertIntoEmptyBranch(...)  — fills an empty parallel branch slot
    deleteNode(node)      — removes a node from the tree

useCircuitModel.ts
  Vue composable. Computes the model impedance trace from the current circuit
  tree and EIS data. Returns ModelData (arrays of Re, Im, freq) for Plotly.

useLMFitting.ts
  Vue composable. Runs Levenberg-Marquardt fitting via lmFitting.worker.ts
  in a background thread. Returns:
    isFitting             — Ref<boolean>  (true while the worker is running)
    fitModel()            — serializes the tree, sends to worker, applies results

--------------------------------------------------------------------------------
  src/components/panels/
--------------------------------------------------------------------------------

DataPanel.vue
  CSV upload and parsing panel (Tab: Data).
  Uses PapaParse (via csvParser.ts) to parse the uploaded file and emits
  'analysis-complete' with the parsed EisDataPoint array and filename.

DataInfoPanel.vue
  Displays statistics about the loaded dataset and runs the Kramers-Kronig
  consistency test via kramersKronig.ts. Shows frequency range, number of
  points, and a pass/fail quality indicator.

PlotPanel.vue
  Nyquist and Bode plots of the raw measurement data (Tab: Plot).
  Uses Plotly.js.

AIPanel.vue
  AI-powered circuit detection panel (Tab: AI).
  Lazily creates an eisInference.worker on the first "Detect ECM" click.
  Displays ranked predictions (circuit string + confidence bar).
  "Apply" button loads the selected circuit into the ECM panel.

--------------------------------------------------------------------------------
  src/components/panels/ECMPanel/
--------------------------------------------------------------------------------

ECMPanel.vue
  The main ECM panel (Tab: ECM). Acts as the top-level glue component:
    - Manages tabs: Builder, Validate, Export.
    - Passes EIS data, AI suggestions, and fit results to child tabs.
    - Coordinates between useCircuitTree, useCircuitModel, and useLMFitting.

ECMBuilderTab.vue
  Circuit builder tab. Hosts the CircuitRenderer (drag-drop canvas),
  CircuitPalette, CommonCircuitsDropdown, and ParameterEditor.
  Shows Nyquist and Bode plots overlaying measured data with the model curve.

ECMValidateTab.vue
  Validation tab. Displays residual plots (ResidualPlot.vue) and chi-squared
  statistics to assess the quality of the fitted model.

ECMExportTab.vue
  Export tab. Allows the user to download fitted parameters and model data
  as CSV files via exportUtils.ts.

CommonCircuitsDropdown.vue
  Dropdown menu populated from commonCircuits.ts. Selecting a circuit loads
  it directly into the ECM builder via buildTreeFromString().

CircuitRenderer.vue  (recursive)
  Renders a CircuitNode tree as an SVG. Recursively renders .next and branch
  nodes. Handles all drag-drop hit areas (before / replace / after zones,
  empty branch drop zones).

CircuitPalette.vue
  Element palette bar. Each item is draggable and sets its element type in
  dataTransfer so CircuitRenderer can receive it.

ParameterEditor.vue
  Renders a grid of labeled number inputs, one per editable node.
  Two-parameter elements (Wo, Ws) receive a second input for τ.

ResidualPlot.vue
  Plots normalised real and imaginary residuals vs. frequency using Plotly.js.
