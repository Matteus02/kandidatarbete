OVERVIEW
--------
A Vue 3 + TypeScript single-page application for analysing EIS measurement
data. The user can upload a CSV file, view Nyquist and Bode plots, let an AI
model identify the equivalent circuit model (ECM), build a circuit manually
using drag-and-drop, adjust parameters, and fit the model to the data.

Technology stack:
  - Vue 3 (Composition API, <script setup>)
  - TypeScript
  - Pinia  (global state store)
  - Plotly.js  (plots)
  - ONNX Runtime Web  (AI inference in a web worker)
  - fmin  (Nelder-Mead optimiser)
  - PapaParse  (CSV parsing)
  - Vite  (build tool)

  src/ai/

modelConfig.ts
  Single source of truth for AI model configuration. Change the model file
  or circuit list here without touching any other file.
    MODEL_URL         — path to the ONNX model file (served from /models/)
    MODEL_EXTERNAL_DATA — path to the .data sidecar file, or null
    KNOWN_CIRCUITS    — the 7 circuit strings the model can predict
    N_POINTS          — number of log-spaced frequency points (60)
    CHANNEL_DEFS      — the 5 input channels fed to the CNN

workerProtocol.ts
  Message types shared between AIPanel.vue and the web worker.
    InferenceRequest  — sent TO the worker: { type: 'run', data: EisDataPoint[] }
    PredictionItem    — { circuit: string, confidence: number }
    InferenceResponse — received FROM the worker: result or error variant

--------------------------------------------------------------------------------
  src/workers/
--------------------------------------------------------------------------------

eisInference.worker.ts
  Runs ONNX model inference in a separate thread so the UI stays responsive.
  Steps performed on each 'run' message:
    1. Sort data high → low frequency.
    2. Interpolate to a fixed 60-point log-spaced frequency grid.
    3. Build a [1, 5, 60] tensor with 5 channels:
         ch0  Re(Z)
         ch1  Im(Z)  (negated: file stores -Im(Z))
         ch2  log10(frequency grid)
         ch3  d(unwrapped phase) / d(log10 f)  — phase derivative
         ch4  d(log|Z|) / d(log10 f)  — Bode slope
    4. Per-channel z-score normalisation.
    5. Run inference via ONNX Runtime Web (WASM backend).
    6. Softmax the logits → confidence scores.
    7. Post { type: 'result', predictions } back to AIPanel.

--------------------------------------------------------------------------------
  src/utils/
--------------------------------------------------------------------------------

complexMath.ts
  Pure complex-number arithmetic. No Vue, no side effects.
  Exports:
    Complex           — type { re: number; im: number }
    add(a, b)         — addition
    mul(a, b)         — multiplication
    div(a, b)         — division
    csqrt(z)          — complex square root (polar form)
    cexp(z)           — complex exponential
    ccosh(z)          — complex hyperbolic cosine
    csinh(z)          — complex hyperbolic sine

impedance.ts
  Maps physical element parameters + angular frequency ω to complex impedance Z.
  Imports complex arithmetic from complexMath.ts.
  Exports:
    parallel(a, b)     — parallel combination Z = (Z1·Z2)/(Z1+Z2)
    zR(R)              — resistor
    zC(C, ω)           — capacitor
    zCPE(Q, n, ω)      — constant phase element
    zW(A, ω)           — Warburg (semi-infinite diffusion)
    zL(L, ω)           — inductor
    zWo(Rw, τ, ω)      — Warburg Open  (finite-length, reflective boundary)
    zWs(Rw, τ, ω)      — Warburg Short (finite-length, transmissive boundary)

circuitImpedance.ts
  Calculates Z(ω) for an entire CircuitNode tree by recursive traversal.
  Imports element formulas from impedance.ts; typed for CircuitNode.
  Exports:
    CPE_N              — fixed CPE exponent (0.85)
    zOfNode(node, ω)   — impedance of a single node or parallel block
    zOfChain(node, ω)  — total series impedance of a chain (follows .next)

circuitParser.ts
  Converts AI circuit strings into CircuitNode trees.
  Input format example:  "R0-p(R1,CPE0)-W0"
    - "-" separates series elements at the top level
    - "p(upper, lower)" describes a parallel block
    - element IDs start with the type letter (R, C, CPE, Wo, Ws, W, L)
  Exports:
    buildTreeFromString(circuitString)  — returns the root CircuitNode

--------------------------------------------------------------------------------
  src/composables/
--------------------------------------------------------------------------------

useCircuitTree.ts
  Vue composable. Owns all circuit tree state and every structural mutation.
  This is the single place where nodes are created, moved, or deleted.
  Returns:
    rootNode              — Ref<CircuitNode>  (the tree root)
    renderVersion         — Ref<number>  (incremented on each change; used as SVG :key)
    collectNodes(root)    — returns all leaf nodes with editable parameters
    resetCounters()       — resets ID counters to 20 (called after AI load)
    handleNodeDrop(...)   — inserts a new node before / replacing / after a target
    insertIntoEmptyBranch(...)  — fills an empty parallel branch slot
    deleteNode(node)      — removes a node from the tree

useCircuitFitting.ts
  Vue composable. Provides parameter estimation and curve fitting.
  Takes rootNode, eisData getter, collectNodes, zOfChain, and an onRedraw
  callback as arguments so it stays decoupled from the DOM.
  Returns:
    isFitting             — Ref<boolean>  (true while the optimiser is running)
    estimateInitialValues()  — reads Nyquist features to seed all parameters:
                               zHF → first R,  arc diameter → parallel R + C/CPE,
                               low-frequency tail → Warburg coefficient
    fitModel()            — Nelder-Mead (downhill simplex) in log-space.
                            Log-space ensures resistances (100s Ω) and
                            capacitances (1e-6 F) have equal numerical weight.
                            Cost function uses modulus weighting (error/|Z|²).

DataPanel.vue
  CSV upload and parsing panel (Tab: Data).
  Uses PapaParse to parse the uploaded file and emits 'analysis-complete'
  with the parsed EisDataPoint array and filename to App.vue.

  Nyquist and Bode magnitude plots of the raw measurement data (Tab: Plot).

  AI-powered circuit detection panel (Tab: AI).

  "Apply" button calls store.setAiSuggestedCircuit() and emits
  'apply-circuit', causing App.vue to navigate to the ECM tab.

ECMmodule.vue
  The main ECM builder module (Tab: ECM). Acts as the "glue" component:
    - Mounts Plotly Nyquist + Bode plots and redraws them when needed.
    - Pulls in useCircuitTree and useCircuitFitting composables.
    - Provides drag-drop handlers to CircuitRenderer via inject/provide.
    - Watches store.aiSuggestedCircuit (immediate: true) to load AI circuits.
    - Delegates palette UI to CircuitPalette, parameter inputs to ParameterEditor.

--------------------------------------------------------------------------------
  src/components/circuit/
--------------------------------------------------------------------------------

CircuitNode.ts
  The core data structure for the circuit tree.
  Class CircuitNode:
    id           — unique string label shown on the symbol (e.g. "R0", "CPE1")
    type         — ElementType: R | C | CPE | W | Wo | Ws | L | parallel | end | empty
    value        — primary parameter (resistance, capacitance, etc.)
    value2       — second parameter used by Wo and Ws (diffusion time constant τ)
    next/earlier — doubly-linked series chain pointers
    upperBranch/lowerBranch — parallel branch child pointers
  Methods:
    countLength() / countMaxLength()  — horizontal pixel span for SVG layout
    countHeight() / countMaxHeight()  — vertical pixel span for SVG layout
    removeNode()                      — detach from the tree

CircuitRenderer.vue  (recursive)
  Renders a CircuitNode tree as an SVG. Recursively renders .next and
  branch nodes. Handles all drag-drop hit areas (before / replace / after
  zones, empty branch drop zones). Injects handleNodeDrop,
  insertIntoEmptyBranch, and deleteNode from the ancestor ECMmodule.

CircuitPalette.vue
  The element palette bar shown below the circuit canvas. Each item is
  draggable and sets its type in dataTransfer so CircuitRenderer can receive
  it. Also shows the "how to use" instructions.

ParameterEditor.vue
  Renders a flex grid of labeled number inputs, one per editable node.
  Two-parameter elements (Wo, Ws) receive a second input for τ.
  Emits 'change' events (unidirectional data flow — does not mutate nodes).

CircuitValue.vue
  Alternative value display component (legacy; not used by ECMmodule).
